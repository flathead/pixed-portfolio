Клиент пришёл с распечатанным списком хотелок на трёх страницах. В самом
низу карандашом: «и чтобы как у Wildberries». Ок.

Начали с аналитики — 3 дня разговоров с командой. Главная боль:
менеджеры вручную обрабатывают 200+ заказов в день через почту.

## Контекст

Клиент — небольшой производитель спортивного питания, работавший через
Wildberries и свой сайт на старом самописном движке. Задача: **переехать
на современный стек** и добавить всё, чего не хватало.

## Проблемы, которые пришлось решать

- **Обработка заказов вручную** — 5 менеджеров в Excel, 200+ заказов в день
- **Сложная система скидок** — персональные, по объёму, купоны, сезонные, и все сразу
- **Интеграция с 1С** — старая система бухгалтерии, которую менять нельзя

## Что сделал

Выбрал **Laravel + Inertia.js + Vue 3** — SPA-ощущение без отдельного API.
Очереди на Redis для уведомлений и генерации документов. Интеграция
с ЮКасса и СДЭК.

Для системы скидок написал отдельный `DiscountEngine` — правила хранятся
в БД, комбинации проверяются через pipeline-паттерн.

## Результат

- Время обработки заказа: с 8 минут до 45 секунд
- 500+ заказов в день без проблем
- Магазин работает по сей день
